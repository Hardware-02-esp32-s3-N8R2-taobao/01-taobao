# 太阳能低功耗电流管理架构

## 推荐架构

```text
太阳能板
  -> 太阳能充电/功率管理模块
  -> 单节锂电池
  -> 高边电源开关 / 受控 3.3V 电源
  -> ESP32 主控
       -> 电池电压检测
       -> 电池电流检测
       -> 电量估算
       -> 传感器采集
       -> Wi-Fi 上传
       -> deep sleep 10 分钟
```

## 建议分层

### 1. 能量层

- 太阳能板
- 充电管理模块
- 锂电池

职责：

- 白天补能
- 晚上放电
- 保护电池

### 2. 电源分配层

- 3.3V 稳压
- 外设电源开关

职责：

- 主控休眠时切断不必要外设
- 尽量让传感器不常上电

### 3. 测量层

- 电池电压
- 电池电流
- 电量估算

职责：

- 看得见当前还剩多少电
- 判断是否需要延长采样周期
- 避免低电状态下频繁联网

### 4. 业务层

- 每 10 分钟唤醒
- 采集传感器
- 上传网络
- 根据电量决定策略

## 监测哪些量

### 电池电压

最低要求：

- 用分压电阻 + ADC

优点：

- 便宜
- 实现最简单

缺点：

- 只能看电压，不等于真实剩余电量

### 电池电流

推荐：

- `INA219` 或 `INA226`

优点：

- 能读电压、电流、功率
- I2C 接口方便

### 电量

推荐分两档：

- 低成本：根据电池电压估算电量
- 更准确：用 `MAX17048` / `LC709203F` 这类 fuel gauge

## 采集周期建议

你当前需求是：

- 每 `10` 分钟采集一次并上传

这个周期是合理的。

建议流程：

1. `RTC` 定时唤醒
2. 上电外设
3. 采集电池电压、电流、电量
4. 采集业务传感器
5. 连接 Wi-Fi
6. 上传数据
7. 断开 Wi-Fi
8. 关闭外设供电
9. 进入 `deep sleep`

## 低电策略建议

建议增加 3 档策略：

### 正常档

- 电压高于阈值
- 每 `10` 分钟上传

### 省电档

- 电压偏低
- 改为每 `30` 分钟上传

### 保护档

- 电压过低
- 只采电压，不联网
- 等太阳能恢复

## 对你当前 ESP32 板的建议

### 原型阶段

可以继续用你现在这块 `DOIT ESP32 DevKit V1`。

### 长期部署阶段

建议换成：

- `FireBeetle 2 ESP32-C6`

原因：

- 官方资料中明确给出太阳能充电和 deep sleep 电流信息
- 对这类项目更友好
