# 成熟方案索引

## 1. 最值得参考的成熟路线

### DFRobot ESP32 太阳能气象站

- 链接：
  - https://mc.dfrobot.com.cn/thread-27416-1-1.html
- 价值：
  - 已经把 `ESP32 + 太阳能板 + 锂电池 + 传感器 + 无线` 组合起来了
- 适合参考：
  - 接线思路
  - 太阳能与电池配比
  - 户外部署形态

### FireBeetle 2 ESP32-C6 官方资料

- 链接：
  - https://wiki.dfrobot.com.cn/_SKU_DFR1075_FireBeetle_2_Board_ESP32_C6
  - https://www.dfrobot.com.cn/goods-3825.html
- 价值：
  - 官方明确写了太阳能充电
  - 官方明确给了 deep sleep 电流
  - 还给了电池电压读取思路

## 2. 更偏工程方法的参考

### solar-esp32

- 链接：
  - https://github.com/fadushin/solar-esp32
- 价值：
  - 适合参考自定义太阳能供电板的思路
  - 包含太阳能板和电池电压监测思路

### esp32-weather-epd

- 链接：
  - https://github.com/lmarzen/esp32-weather-epd
- 价值：
  - 很适合参考低电压保护策略
  - 有“低电时延长休眠周期”的思路

### esp32_ttn_environmental_sensor

- 链接：
  - https://github.com/JackGruber/esp32_ttn_environmental_sensor
- 价值：
  - 虽然走的是 LoRaWAN，但低功耗传感节点的系统组织方式很值得参考

## 3. 我建议你怎么参考

如果你是要做：

- 太阳能供电
- 电池电压 / 电流 / 电量监测
- 10 分钟一次上传

建议参考顺序：

1. `FireBeetle 2 ESP32-C6` 官方资料
2. `DFRobot ESP32 太阳能气象站`
3. `esp32-weather-epd`
4. `solar-esp32`

## 4. 关键经验

成熟方案里普遍有几个共同点：

- 不会让 Wi-Fi 常开
- 会大量使用 `deep sleep`
- 会在低电压时降低上传频率
- 会尽量减少板级待机损耗
