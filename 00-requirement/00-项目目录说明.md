# 项目目录说明

本文档用于说明当前项目的一级目录分布。当前仓库根目录按功能收敛为 4 个主要文件夹：

- `00-Requirement`：需求、协议、架构与目录说明
- `01-firmware`：所有固件相关内容
- `02-app`：电脑端上位机
- `03-server`：网页服务端与网页入口

## 1. 根目录结构

```text
YD-ESP32-S3/
├── 00-Requirement/
├── 01-firmware/
├── 02-app/
└── 03-server/
```

## 2. 各目录职责

### 2.1 `00-Requirement`

主要保存正式需求和说明文档，包括：

- 当前需求文档
- 网页架构文档
- App 架构文档
- 通信协议说明

当前建议把所有“协议约定、需求变更、目录规划、设计说明”都直接沉淀到正式编号文档中，不再额外保留参考资料和会话记录副本，避免内容分散和重复。

### 2.2 `01-firmware`

主要保存固件相关工程，其中：

- `00-main-c3-firmware`：当前主固件开发目录
- `10-esp32-s3-other`：其他 ESP32-S3 相关资料与工程
- `11-esp32-other`：其他 ESP32 相关资料与工程
- `12-esp32-c3-other`：其他 ESP32-C3 资料与历史工程

主固件目录中额外提供：

- `build_firmware.py`：用于编译固件
- `flash_firmware.py`：用于烧录固件

### 2.3 `02-app`

主要保存电脑端上位机内容：

- `01-c3-monitor`：当前 C3 串口监控上位机

目录中额外提供：

- `启动上位机.py`：用于启动上位机
- `启动上位机.bat`：用于桌面快捷方式直接启动

### 2.4 `03-server`

主要保存网页服务端内容：

- `01-net-display-server`：当前网页服务端与页面代码

目录中额外提供：

- `打开网页.py`：用于打开网页入口
- `打开网页.bat`：用于桌面快捷方式直接打开网页

## 3. 当前整理原则

1. 固件、App、Server 各自独立，避免功能混放。
2. 需求、协议、规划、历史记录统一放在 `00-Requirement`。
3. 桌面直接使用的启动入口优先放在 `02-app` 和 `03-server` 根目录。
4. 主开发工程优先放在每类目录中最容易找到的位置。
