# 硬件接线说明 — ESP32-C3 SuperMini

> 本文档描述 `02-wifi-mqtt-forward-demo` 工程中各硬件设备的实际接线方式，以及新增设备时推荐使用的 IO 口。
>
> 对应工程：`03-esp32-c3/03-project/02-wifi-mqtt-forward-demo`

---

## 一、当前已接设备

### DHT11（温湿度传感器）

**接口类型：** 单总线（Single-Wire）

| DHT11 引脚 | 接到 ESP32-C3 |
|-----------|--------------|
| VCC       | 3V3          |
| GND       | GND          |
| DATA      | **GPIO4**    |

- 代码定义位置：`main/include/app_config.h` → `APP_DHT11_GPIO GPIO_NUM_4`
- GPIO4 内部已启用上拉，无需外接上拉电阻

---

### 板载 LED

| 说明     | GPIO       |
|---------|-----------|
| 板载 LED | **GPIO8** |

- 低电平点亮（active-low）

## 二、已有驱动但当前未接的设备

### OLED SSD1306（128×64 I2C 显示屏）

**接口类型：** I2C（`I2C_NUM_0`）

> ⚠️ 注意：`01-led-oled-demo` 工程中 OLED 默认使用 `GPIO4(SDA)` + `GPIO5(SCL)`，
> 但 `GPIO4` 已被 DHT11 占用。
> 在本工程（`02-wifi-mqtt-forward-demo`）中，如果要接 OLED，**必须改用以下引脚**：

| OLED 引脚 | 接到 ESP32-C3        |
|----------|---------------------|
| VCC      | 3V3                 |
| GND      | GND                 |
| SDA      | **GPIO6**（推荐）    |
| SCL      | **GPIO7**（推荐）    |

- I2C 地址：自动探测 `0x3C` / `0x3D`（驱动内部已支持）
- 启用方法：在 `app_config.h` 中增加以下宏，并在 `app_main.c` 中调用 `oled_ssd1306_init()`

```c
#define APP_OLED_SDA_GPIO    GPIO_NUM_6
#define APP_OLED_SCL_GPIO    GPIO_NUM_7
#define APP_OLED_I2C_PORT    I2C_NUM_0
#define APP_OLED_CLOCK_HZ    400000
```

## 三、新增设备推荐使用的 IO 口

ESP32-C3 SuperMini 可用 GPIO 一览（已排除占用引脚）：

| GPIO | 当前状态            | 推荐用途              |
|------|--------------------|--------------------|
| GPIO0 | 空闲              | 通用 IO / ADC       |
| GPIO1 | 空闲              | 通用 IO / ADC       |
| GPIO2 | 空闲              | 通用 IO / ADC       |
| GPIO3 | 空闲              | 通用 IO / ADC       |
| GPIO4 | **DHT11 DATA**    | 已占用              |
| GPIO5 | 空闲              | 通用 IO             |
| GPIO6 | 空闲（OLED SDA 推荐）| I2C SDA           |
| GPIO7 | 空闲（OLED SCL 推荐）| I2C SCL           |
| GPIO8 | **板载 LED**      | 已占用              |
| GPIO9 | BOOT 按键（默认上拉）| 可复用但需注意启动    |
| GPIO10 | 空闲             | 通用 IO             |
| GPIO20 | 空闲             | 通用 IO / UART RX  |
| GPIO21 | 空闲             | 通用 IO / UART TX  |

## 四、新增传感器接线快查表

### 单总线类（如 DHT11、DS18B20）

接一根 DATA 线到任意空闲 GPIO，在 `app_config.h` 中定义对应宏即可。

| 传感器    | 推荐接法            |
|----------|-------------------|
| DHT11    | DATA → GPIO4（已接）|
| DS18B20  | DATA → GPIO5（推荐）|

### I2C 类（如 OLED、BMP280、SHT30、AHT20）

同一组 I2C 总线（SDA + SCL）上可以挂多个设备（地址不同即可）。

| 总线     | SDA       | SCL       | 可挂设备示例                     |
|---------|-----------|-----------|-------------------------------|
| I2C_NUM_0 | **GPIO6** | **GPIO7** | OLED SSD1306、BMP280、SHT30、AHT20 |

> 同一组 I2C 总线上挂多个器件时，只需共用 GPIO6 和 GPIO7，无需重复初始化总线。

### ADC 类（如土壤湿度、光敏电阻、MQ 系列气体传感器）

ESP32-C3 的 ADC1 通道对应 GPIO：

| ADC 通道  | GPIO    |
|----------|---------|
| ADC1_CH0 | GPIO0   |
| ADC1_CH1 | GPIO1   |
| ADC1_CH2 | GPIO2   |
| ADC1_CH3 | GPIO3   |
| ADC1_CH4 | GPIO4   |（已被 DHT11 占用）

推荐使用 GPIO0～GPIO3 接模拟量传感器的信号输出端。

## 五、供电说明

| 电源引脚 | 说明                    |
|---------|------------------------|
| 3V3     | 板载 3.3V 稳压输出，最大约 600mA |
| 5V      | USB 供电直通，约 500mA（由 USB 决定）|
| GND     | 所有设备共地            |

- DHT11、OLED SSD1306、BMP280 均使用 3V3 供电
- 若同时接多个 I2C 设备，建议共用 3V3 和 GND
