> 创建日期：2026-03-18
> 项目目录：`03-esp32-c3`

---

## 一、项目目标

构建一套**"烧录同一份固件、App 配置身份、云端按设备展示数据"**的通用传感器上报平台。

核心特性：
- 同一份固件可批量烧录到多台 ESP32-C3 设备
- 首次上电通过 App（BLE 配网）配置设备名和 Wi-Fi 凭据
- 设备通过 MQTT 上报传感器数据，Topic 中携带设备名
- 服务端/网页根据设备名动态归类展示数据
- 固件通过插件化传感器驱动，按需开启不同传感器类型

- **框架**：ESP-IDF（推荐）或 Arduino + ESP-IDF 组件

### 3.3 传感器插件机制

每个传感器驱动实现统一接口，通过注册宏挂载到调度器：

```c
// sensor_interface.h
typedef struct {
    const char *type;                          // 传感器类型名，如 "dht11"
    esp_err_t (*init)(void);                   // 初始化
    esp_err_t (*read)(char *json_buf, int len); // 读取并填充 JSON 字段
} sensor_driver_t;

// 注册宏（放在驱动 .c 文件末尾）
#define REGISTER_SENSOR(drv) \
    __attribute__((used, section(".sensor_registry"))) \
    static const sensor_driver_t *_reg_##drv = &drv;
```

### 3.4 MQTT Topic 设计

```
上报数据：  sensor/{device_name}/{sensor_type}
示例：      sensor/living-room-01/dht11
            sensor/garden-02/bmp280

设备状态：  device/{device_name}/status
OTA 指令：  device/{device_name}/ota
配置下发：  device/{device_name}/config
```

数据格式（JSON）：
```json
{
  "device": "living-room-01",
  "type": "dht11",
  "ts": 1710000000,
  "data": {
    "temperature": 25.3,
    "humidity": 60.1
  }
}
```

### 3.5 设备状态机

```
上电
 │
 ▼
读取 NVS 配置
 │
 ├─ 无配置 ──▶ 开启 BLE 广播，等待 App 配网
 │                    │
 │             配网成功，写入 NVS
 │                    │
 └─ 有配置 ◀──────────┘
         │
         ▼
      连接 Wi-Fi
         │
         ▼
      连接 MQTT
         │
         ▼
      初始化已注册传感器
         │
         ▼
      定时采集 + 上报（主循环）
```

## 四、手机 App 架构

### 4.1 技术选型
- **框架**：Flutter（iOS + Android 一套代码）或 uni-app
- **BLE 库**：`flutter_blue_plus`（Flutter）
- **本地存储**：sqlite / shared_preferences（保存已配置设备列表）

### 4.2 功能模块

```
app/
├── lib/
│   ├── pages/
│   │   ├── scan_page.dart        # BLE 扫描页：发现未配置设备
│   │   ├── provision_page.dart   # 配网页：输入设备名、Wi-Fi、MQTT 地址
│   │   ├── device_list_page.dart # 已配置设备列表
│   │   └── device_detail_page.dart # 单设备传感器数据实时查看
│   ├── ble/
│   │   └── ble_service.dart      # BLE GATT 通信封装
│   ├── models/
│   │   └── device_model.dart     # 设备数据模型
│   └── main.dart
```

### 4.3 BLE 配网流程

```
App 扫描 BLE 广播（ServiceUUID 过滤）
         │
         ▼
用户选择设备 → 连接
         │
         ▼
写入 Characteristic：
  - device_name: "living-room-01"
  - wifi_ssid:   "MyWiFi"
  - wifi_pass:   "password"
  - mqtt_host:   "192.168.1.100"
         │
         ▼
ESP32 断开 BLE，尝试连接 Wi-Fi + MQTT
         │
         ▼
App 收到配网结果通知（success / fail）
```

### 4.4 App 配置项（可配置内容）

| 配置项 | 说明 |
|--------|------|
| 设备名 | 全局唯一，用于 MQTT Topic 和网页标识 |
| Wi-Fi SSID / 密码 | 设备所在网络 |
| MQTT Broker 地址 | 服务器 IP 或域名 |
| MQTT 端口 | 默认 1883 |
| 采集间隔 | 传感器上报频率（秒） |
| 启用传感器列表 | 勾选本设备挂载的传感器类型 |

## 五、后端服务架构

### 5.1 技术选型
- **运行时**：Node.js（延续现有 `04-server/05-new-net-display`）
- **MQTT 客户端**：`mqtt.js`
- **实时推送**：Socket.IO（WebSocket）
- **数据存储**：SQLite（轻量）或 InfluxDB（时序数据，可选）
- **HTTP 框架**：Express.js

### 5.2 服务模块

```
server/
├── src/
│   ├── mqtt/
│   │   └── mqtt_subscriber.js   # 订阅 sensor/# 主题，解析并分发
│   ├── db/
│   │   ├── device_store.js      # 设备注册表（设备名 → 位置/分组映射）
│   │   └── data_store.js        # 传感器历史数据存储
│   ├── ws/
│   │   └── socket_server.js     # Socket.IO 推送实时数据给网页
│   ├── api/
│   │   └── router.js            # REST API：设备列表、历史数据查询
│   └── server.js                # 入口
├── package.json
└── .env                         # MQTT_HOST, DB_PATH 等配置
```

### 5.3 数据流

```
ESP32 → MQTT Broker → mqtt_subscriber.js
                              │
              ┌───────────────┼────────────────┐
              ▼               ▼                ▼
         写入数据库      按设备名分组       Socket.IO 推送
                                               │
                                               ▼
                                           网页实时更新
```

## 六、网页端架构

### 6.1 技术选型
- **框架**：Vue 3 + Vite（或 React）
- **UI 组件**：Element Plus / Ant Design Vue
- **图表**：ECharts（实时折线图、仪表盘）
- **实时通信**：Socket.IO client

### 6.2 页面结构

```
web/
├── src/
│   ├── views/
│   │   ├── Dashboard.vue        # 所有设备总览（卡片式布局）
│   │   ├── DeviceDetail.vue     # 单设备详情（按传感器类型分 Tab）
│   │   └── DeviceManager.vue    # 设备管理（设置设备显示名、分组、位置）
│   ├── components/
│   │   ├── SensorCard.vue       # 通用传感器数据卡片
│   │   ├── RealtimeChart.vue    # 实时折线图组件
│   │   └── DeviceStatusBadge.vue # 在线/离线状态
│   ├── store/
│   │   └── device_store.js      # Pinia 状态管理
│   └── socket/
│       └── socket.js            # Socket.IO 连接封装
```

### 6.3 网页展示逻辑

- 网页通过 Socket.IO 接收实时数据，key 为 `device_name`
- 根据 `device_name` 定位到对应的设备卡片并刷新数值
- 支持按**分组/房间/位置**过滤设备
- 每种传感器类型有对应的展示组件（温度计、湿度圆环、折线图等）
