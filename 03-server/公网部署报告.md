# 公网部署报告

生成时间：2026-03-22

## 一、部署概述

服务完全部署在云服务器上，通过 Nginx 反向代理提供公网 HTTP 服务，无内网穿透、无 Cloudflare Tunnel、无树莓派。

| 项目 | 值 |
|------|-----|
| 公网地址 | `http://117.72.55.63/` |
| 服务器 | 京东云 VPS，Ubuntu 24.04.2 LTS |
| Node.js | v22.22.1 |
| 部署目录 | `/opt/yard-display` |
| 服务管理 | systemd `yard-display.service` |
| 反向代理 | Nginx（`/etc/nginx/sites-enabled/default`） |

---

## 二、实际数据链路

```
ESP32 传感器（任意网络）
  └─ MQTT TCP → 117.72.55.63:1884
                 └─ Aedes MQTT Broker（内嵌于 Node.js）
                     └─ 写入 SQLite 数据库

用户浏览器（任意公网位置）
  └─ HTTP → 117.72.55.63:80（Nginx）
              ├─ /api/weather/forecast → [Nginx 缓存 25 分钟] → 127.0.0.1:3000
              └─ 其余所有请求 → 127.0.0.1:3000（Node.js）

Node.js（127.0.0.1:3000）
  ├─ 服务前端 HTML / JS 静态文件
  ├─ 提供 REST API（传感器数据、设备状态、服务器监控）
  ├─ 拉取杭州天气（Open-Meteo 免费 API，30 分钟缓存）
  └─ 读写 SQLite 数据库（/opt/yard-display/data/sensor-history.db）
```

---

## 三、端口说明

| 端口 | 协议 | 绑定地址 | 对外暴露 | 用途 |
|------|------|----------|----------|------|
| **80** | HTTP | `0.0.0.0` | **是** | Nginx 反向代理入口（网页 + API） |
| **3000** | HTTP | `0.0.0.0` | 否（仅本机） | Node.js 服务（Nginx 内部转发） |
| **1884** | MQTT TCP | `0.0.0.0` | **是** | ESP32 传感器数据接入（Aedes Broker） |

> 端口 3000 虽绑定 `0.0.0.0`，但云服务器安全组通常默认仅开放 80/22，实际外部访问由 Nginx 代理。

## 四、服务器配置

### systemd 服务（`/etc/systemd/system/yard-display.service`）

```ini
[Unit]
Description=Yard Display - Smart Home Dashboard
After=network.target mosquitto.service
Wants=mosquitto.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/yard-display
Environment=PORT=3000
Environment=MQTT_PORT=1884
ExecStart=/usr/bin/node server.js
Restart=always
RestartSec=5
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

### Nginx 配置要点（`/etc/nginx/sites-enabled/default`）

- 所有 `location /` → `proxy_pass http://127.0.0.1:3000`
- `location = /api/weather/forecast` → 额外 Nginx 缓存 25 分钟
- `location /mqtt` → `proxy_pass http://127.0.0.1:9001`（Mosquitto WebSocket，系统级服务）

## 八、注意事项

1. **MQTT 端口 1884 无鉴权**：当前 Aedes Broker 无用户名/密码，任何能访问该端口的设备均可发布数据。如需安全性，可为 Aedes 添加认证中间件。

2. **HTTP 无 HTTPS**：当前使用 HTTP，数据传输未加密。如需 HTTPS，可申请免费证书（Let's Encrypt）并配置 Nginx SSL。

3. **水泵密码**：默认密码 `1234`，建议通过服务器环境变量 `PUMP_CONTROL_PASSWORD` 修改：
   ```bash
   systemctl edit yard-display
   # 添加 Environment=PUMP_CONTROL_PASSWORD=your_password
   ```

4. **数据库备份**：`/opt/yard-display/data/sensor-history.db` 未做自动备份，建议配置定时任务。

5. **内存余量偏低**：服务器总内存 1.92 GB，已用 1.42 GB，建议关注内存使用趋势，避免 OOM。
